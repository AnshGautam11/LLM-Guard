

1) Explain about HTTP and HTTPS difference
Ans :- HTTP and HTTPS are both protocols for transferring data on the web, but HTTPS adds encryption and authentication,
       making it much more secure than HTTP.
	   
	 # What HTTPS Means
     1] HTTPS stands for HyperText Transfer Protocol Secure , and it is essentially HTTP plus a security layer called TLS/SSL.
     2]HTTPS encrypts the data between browser and server, so even if someone intercepts it, they cannot easily read or alter it.
	 3]HTTPS usually uses port 443,and URLs start with https://; modern browser show a padlock icon for HTTPS sites.
	 
	  # What is HTTP 
	 1] HTTP stands for HyperText Transfer Protocol and defines how browsers and servers communicate to load web page, image , vedio 
	    and other resources.
	 2] Data in HTTP is sent as plain text , which means anyone who intercepts the traffic can read or modify it.
	 3] HTTP typically uses port 80 on the server, and URLs start with http://
	
  ## Mein Differences
    1] Security : HTTP has no encryption or built-in authentication, while 
                  HTTPS uses TLS/SSL to provide encryption, server identity verification,and protection against man-in-the middle attacks.
    2] USE CASE : HTTP is suitable only for non-sensitive content, where as HTTPS is strongly recommended (and often required) for login pages,
                  payments,APIs, and almost all modern websites.

 3)what are the stages of the vulnerability lifecycle?
 Ans:- The vulnerability lifecycle is usually described as five main stages :- 1.Discovery, 2.Coordination and Disclosure, 3.Mitigation, 4.Deployment/Management,
       Lessons ,Learned/continuous improvement.
	   
	   1]Discovery:- A vulnerability is first discovered in software, configuration,or the environment by researcher, vendor, attacker,or automated scanning tool.
	   2]Coordination & Disclosure:- The finder typically coordinates so they can understand and reproduce the issue.
	                                 This may invovle resonsible disclosure or public disclosure vai advisories,bug trackers,or vulnerability databsed such as CVE.
	   3]Mitigation :- The vendor or system owner designs and implemnts a mitigation strategy, often in the form of patches, configuration changes or comensting controls. 							 
	   4]Deployment and Ongoing Management:- Organiztions must then deploy the patch or other mitigations across all affected assets, verify success, and track status .
	   5]Lessons Learned and Continuous Improvement:- Finally, organiztions analysze what happened, how quickly they responded, and what can be improved in processes, tooling and traning.

7) what is CVE and what information does it provide?
Ans:- CVE (Common vulneribility Exposures) is a global naming and cataloging system for publicly known security vulnerabilities and exposures in software and hardware.
     Each CVE entry gives a standardized "record" so vendors,tools,and defenders can all talk about the same issue

   # What information a CVE entry provides
   1] CVE ID: A unique identifier in the from CVE-YYYY-NNNNN (year plus sequence number),used in patch notes,scanner outputs, and advisories.
   2] Brief description :- A short text describing the vulnerable product, affected versions, and the nature of the vulnerability
   3] Referenaces:- Links to vendor advisories, security researcher write-ups, patches, or other sources with more technical detail and remediation infornatuons.   
  
8)what is an Insecure Direct Object Reference (IDOR)?
Ans:-  An Insecure Direct Objectb Reference(IDOR) is an access control vulnerability where an application lets a user directly refernce an internal object 
      (like a record ID, filename,or user ID) and fails to properly check whether that user is authorized to access that specific object. In practice,it
	  means you can often change an ID in URL OR request parameter and gain access to someone else data or actions.

10)what is an auxiliary module in Metasploit?
Ans:- An auxiliary module in Metasploit is any module that is not designed to deliver a payload or "pop a shell:,but instead performs
      supporting tasks like scanning, enumeration, fuzzing,brute forcing, or other non-exploit actions. In short, its a "utility" module used for information 
	  gathering and assessment rather than direct exploitation.
